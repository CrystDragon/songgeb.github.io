iOS中URL使用注意事项

发表于 2018-01-18 | 分类于 Swift | 阅读次数:

NOTE: 本文内容基于Swift 4版本

URL格式 scheme : // net_loc / path ; params ? query # fragment

正常使用URL的姿势：

fileURLWithPath

init?(string: String)

总结

  1. fileURLWithPath一般用于本地文件
  2. URL类(Swift)无法识别URL中的params部分，Objective-C中是可以的
  3. fileURLWithPath会自动进行percentEncoding。而init?(string: String)则不会

疑问

如果用urlsession对本地文件fileURL发起请求会怎样

会正确返回数据，但回调执行在子线程中

为什么FilemManager使用url.absoluteString不work？而是得用url.path

此处使用了absoluteString, 首次会执行if块中的代码创建dirURL，但之后重复运行到if判断，还是会进入创建路径的语句。原因是fileExists接收path的参数，而不是一个url的字符串。此处改为dirURL.path则正确。而且filemanager很多方法的声明中会通过atPath还是atURL来区分接收的参数类型。
