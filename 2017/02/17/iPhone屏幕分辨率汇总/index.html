生成universal framework

使用lipo命令将模拟器和真机的framework和为可共用的版本

lipo -create -output {输出路径} {framework1} {framework2}

  1. 注意此处的framework1和2指的是framework/目录下面的以productname为命名的可执行文件
  2. 输出结果也是一个可执行文件，需要和framework其他部分组合到一起才可以使用

较早的WWDC视频没字幕怎么办

Youtube支持为视频自动生成字幕，多种语言都可以。机器学习的牛逼之处。仅需两步即可实现为视频加字幕：

  1. 把wwdc视频下载到本地
  2. 上传到Youtube上，就会自动生成了。（如果视频时间比较长可能要多等一会儿，youtube要完整的分析一遍上传的视频）

OTA(over the air)方式发布测试包(ipa)

简言之，OTA方案可以使安装ipa测试包无需数据线，直接点击网页中的链接，ipa就安装到手机上了。

该方案由苹果官方提供，很早就有了。关键在于iOS的webview支持itms-services协议，再配合一个描述安装包(ipa文件)的plist文件就可以搞定。

一个网页链接(遵守itms-services协议) + 配置信息plist文件 + 支持https的server

具体配置过程可以参考:iOS通过OTA方式分发应用的实现 和 iOS 7.1在线安装IPA失败以及数字证书

NOTE:
配置过程有一个坑是，由于iOS要求plist文件的是在https服务器上，并且ssl证书要求得是权威机构认证的而不是私人生成的证书。这里无需自费购买ssl证书，github.com支持ssl，可以将plist文件放到github上。ipa放到非https服务器上是没问题的。

解决思路

基本解决思路就是：

  1. 查看官方支持的音视频格式
  2. 查看视频的本身的各项参数是否符合1中的要求

关于dispatch_once中onceToken

dispatch_once_t 的定义是

  1. 创建单例时，系统根据该值判断block中代码是否已执行。所以，要求该值不能动态产生。生命周期要是整个app。所以通常是static的。
  2. GCD这种单例写法相比@synchronized加锁的写法性能要高

更多内容可以参考GCD实践（三）使用dispatch_once来执行只需运行一次的线程安全代码

打包ipa的另一种方法

由于最近公司的苹果证书被锁定，再加上Xcode中登录的apple id会话过，这就导致在用自己机器打包(archive)的时候，最后选择打包方式的时候（如图），选择任何一种方式导出时，都会提醒账户已锁定无法导出。

另一种导出ipa的方法是：

  1. 使用正确的证书（开发者证书、企业发布证书都可以）执行build
  2. 打开iTunes -> 应用选项，将build出的.app文件拖拽到iTunes中
  3. 在iTunes中选择刚才拖过去的app，右键 -> 在finder中显示，就找到最终的ipa文件了

亲测，对于企业发布证书和开发者证书打的包都适用。
