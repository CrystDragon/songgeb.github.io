要解决的问题

该篇技术贴主要解释iOS怎样加载高分辨率的图片(@2x、@3x)

干货

目前(2017年08月09日)，iOS开发中需要提供三种分辨率的图片分辨是1、2、3倍图。
(关于哪些设备是1/2/3倍分辨率可以参考我的另一篇帖子)

命名规则：

  • Standard: <ImageName><device_modifier>.<filename_extension>
  • High resolution: <ImageName>@2x/@3x<device_modifier>.<filename_extension>

device_modifier这个是可选的，它指明了设备信息，比如~ipad、~iphone。
必须按照此顺序，否则系统找不到图片。

系统如何加载图片

  1. imageNamed:
  2. imageWithContentsOfFile:
  3. initWithContentsOfFile:

Note：

  • 使用上面三个方法，系统会根据当前设备分辨率寻找合适的图片。
  • 如果相应分辨率的图片没找到，系统会找最近的低分辨率的图片。即@3x > @2x > @1x顺序
  • 获取的UIImage对象的scale属性会根据不同分辨率自动设置为合适的值

其他

加载UIImage相关问题还有很多，比如使用Core Graphics画出的图片需要对scale做处理、UIBezierPath画出的线宽度在不同分辨率显示的处理等。
由于目前还未接触到，先不深究。详情可以参考文章最后的链接。

iOS应用跳转到app store应用的方法

发表于 2017-02-08 | 分类于 原创 | 阅读次数:
  • UIWebView加载链接跳转到App Store/iTunes Store
  • openURL打开appstore相关链接

本文代码测试环境是
Xcode 8.0 + iPhone 5(iOS10系统)

UIWebView加载链接跳转到App Store/iTunes Store

当UIWebView加载以下几种链接格式的链接时会自动跳转到App Store或iTunes Store

  1. 完整链接(两种形式)

    成功跳转到App Store，webview should load执行两次，且无load start和load结束的回调。

    • https://itunes.apple.com/us/app/apple-store/id375380948?mt=8

      两次加载的URL分别为：

      1. https://itunes.apple.com/us/app/apple-store/id375380948?mt=8
      2. itms-appss://itunes.apple.com/us/app/apple-store/id375380948?mt=8

    • https://itunes.apple.com/app/id378458261

      两次加载的URL分别为：

      1. https://itunes.apple.com/app/id378458261
      2. itms-appss://itunes.apple.com/app/id378458261

  2. App Store Short Links(即短链接)

    • https://appstore.com/applestore

      webview should load方法被调用三次，最终调到App Store的相应app页面，但最终load fail方法调用一次(错误信息是load frame interrupt)。三次加载的URL分别是：

      1. https://appstore.com/applestore
      2. https://search.itunes.apple.com/WebObjects/MZContentLink.woa/wa/link?mt=8&path=apps%2fapplestore
      3. itms-appss://search.itunes.apple.com/WebObjects/MZContentLink.woa/wa/link?mt=8&path=apps%2fapplestore

      由于

      Previously, short links were available using the itunes.com URL. AppStore.com replaces itunes.com. iTunes.com links will continue to work but should be updated as soon as feasible.

      所以，有可能如下的形式可以跳转:

    • https://itunes.com/applestore(经测试该链接跳转到itune store应用,且跳转很慢)

  3. itms-apps/itms-appss

    该类链接跳转时UIWebView执行进行一次请求便自动跳转到App Store相应app页面，无加载结果回调。

    • itms-apps://itunes.apple.com/app/id378458261
    • itms-appss://itunes.apple.com/app/id378458261

  4. itms

    执行一次请求直接跳转到iTunes Store应用的app页面，无回调信息

    • itms://itunes.apple.com/app/id378458261

openURL打开appstore相关链接

使用[[UIApplication sharedApplication] openURL:];直接打开链接

  1. 完整链接(官方推荐)

    直接跳转到App Store相应应用页面

    • https://itunes.apple.com/app/id378458261
    • https://itunes.apple.com/us/app/apple-store/id375380948?mt=8

  2. 短链接

    • https://appstore.com/applestore

      先跳转到safari，留下空白页，再跳转到App Store

    • https://itunes.com/applestore

      先跳转到safari，留下空白页，再跳转到iTunes Store

  3. itms

    直接跳转到iTunes Store

    • itms://itunes.apple.com/app/id378458261

  4. itms-apps/itms-appss

    直接跳转到App Store

    • itms-apps://itunes.apple.com/app/id378458261
    • itms-appss://itunes.apple.com/app/id378458261

目前hexo editor已经转移到搬瓦工上买的一个vps空间里了

2017-02-11 17:53:39
加入文章源文件自动备份功能

背景

之前在CSDN、作业部落、简书都写过文章，主要用作技术备忘和分享。现在想自己搭建一个博客，主要解决以下几个需求：

  1. 支持markdown语法（大部分第三方博客系统都支持）
  2. 移动端也可以轻松写文章（简书和CSDN有app，作业部落不可以）
  3. 支持更多自定义选项，比如加入广告位
  4. 最重要的一点，the quicker the better! 搭建过程越简单越好，我希望尽量把时间用在文章内容上

初步方案

看了一些自己搭建博客的文章，选择了Hexo开始搞。大概原理是：

  1. 文章内容（md文件）结合网页的风格等配置信息，转换为网页
  2. 网页可部署到本地或远端服务器（我选择用github的page服务，其他像coding.net也支持）

发现有以下几个局限性：

  1. 为了更好的写作体验，需要自己安装markdown编辑器支持预览等功能
  2. 由于博客文章的源文件（md文件）和转换后的网页是分离的，网页会上传到git服务器上，源文件则在本地。这样不利于在不同机器上写文章，目前有些做法是同步不同机器的源文件，不管使用工具同步还是放到git服务器上同步，我表示都不太能接受
  3. 只能使用命令来进行各种操作，没有可视化界面

最终方案

Hexo + 自动备份源文件 + Hexo Editor + vps

  • Hexo Editor

    相当于Hexo的Web IDE，nodejs编写，支持在线文章编辑、预览、转换为网页、部署等功能；并且对移动端做了适配 github地址

  • VPS(虚拟专用服务器)

    提供服务器

  • 自动备份源文件

    为hexo加入源文件自动备份机制，基本原理是监听Hexo部署结束的方法，部署成功后通过git的shell命令对vps服务器上的博客源文件进行自动备份。详细内容可参考下面的链接。

我就不写具体教程了，网上很容易查到，可查看参考链接，大体步骤如下：

  1. Hexo博客源文件和Hexo Editor都部署到vps上
  2. 加入自动备份源文件脚本以及其他插件

  3. 开启Hexo Editor服务

科学生活

发表于 2017-01-29 | 分类于 思考 | 阅读次数:

我将在我的博客中时不时地记录一些我认为是科学的生活tips。互联网时代，尽管信息多了，但人们做选择更难了，你认为你在微信上能选择看到更多好文章了？其实你是被那些文章吸引着，它让你看你就看，让你转你就转，你是在选择它吗？我们的生活被这些互联网产品碎片化，你以为增长知识了，但你如何甄别你看到的文章不是几个聪明人写的策划案，你只是被操控帮他们造势而已。比如“民科”郭英森，这真是民间科学牛人被学术权威打压的故事吗？他要造永动机和长生不老药，这些你知道吗？

缘起

《罗辑思维》到现在已经几百集了，我很自豪自己从头到尾一集也没落地看完了，而且都是主动去看。现在它已经是我学习生活、科学知识的一个重要途径了。

从《罗辑思维》中我学到了科学地看待中医，学到了拒绝逃离北上广，学到了不要违背人性的欲望，应该顺从它并想办法实现自己的目标，还有好多。

推荐

除了《罗辑思维》，再推荐几个的靠谱的知识来源，《卓老板聊科技》，《精英日课》。

干货

如果这篇文章的读者你疑问，我凭什么理直气壮地列出这些东西并且保证是对的。我的答案是，我不能保证是对的，科学本身就不能保证一切正确，而是通过严谨的反复的可验证真伪的科学实验，论证推导并解释各种现象，不断推陈出新，科学发展是无限接近正确，仅此而已。我作为一个普通人，学习科学的生活方式不可能自己去研究各种专业知识，我只能选择相信我认为会坚持科学思想的那群人，也就是《罗辑思维》、《卓老板聊科技》等，仅此而已。

生活

  1. 科学实验证实，胎教音乐对胎儿并没有提升智力等各种能力的作用。
  2. 甩脂机器被证明对减肥并无作用。
  3. 失眠怎么办？科学界难题，没有根本解决办法。一般正常情况，做到以下四点会有明显好转：（1）按时作息，避免晚睡晚起，尽量达到7小时睡眠，即使是周末，即使早上很困不想起。我过去也认为这是废话，但当工作的压力和蛋疼的精神困扰你很久后，你会妥协的。（2）床只用来睡觉，避免在床上工作、娱乐等。（3）如果躺下很长时间仍然睡不着，那就不要逼自己了，起来干点别的事。（4）经常运动。如果这些都不起作用，不要自己当医生，找专业医生去吧。
  4. 大城市开车上班堵车问题严重，一个不错的解决办法是：轻客，一种助力电单车，续航骑行40-60公里左右，根据当前路面情况、风阻智能调整动力、扭矩等，还有像蓝牙连接、gps定位之类有意思的功能。很重要的一点是，相比于普通电动车，重量轻，车身尺寸小，可以带上地铁，遇到堵车抬着就上人行道了。目前有两款，售价3000+。官网
  5. 新生儿脐带血要不要保存？不要。保存脐带血是从2000年由国外引入，脐带血中有未分化的造血干细胞，如果将来得白血病或一些免疫疾病，可以通过诱导之前保存的脐带血，生成和自己体内一摸一样的血液系统和免疫系统。但先等等，2008年做过一次统计，从2000年开始全球保存新生儿脐带血的数量是55万多例，期间被起用过的仅99例，最后成功有效的只有1例，有效比例是1/55万。其实像白血病之类本身发病就很低，而且脐带血仅能保存十几年，虽然一人保存脐带血只需要两万多块钱，但从长远来看，这等同于你得了一个绝症，有一种新药有1/100的可能治愈你的疾病，但价格是1亿人民币，你是否愿意花这个钱？人们听说脐带血其实大多源自医院机构发的宣传单，这并非是个可以造福全民的高价值的技术，而是商业机构夸大宣传，把实际的成本拆分为人们可接受的价格，却不告知极低的有效率的手段，对他们来说是一本万利的。所以完全没必要保存，除非你愿意花那一个亿。
  6. 多吃味精会有害吗？没有害处。味精的主要组成成份是谷氨酸钠，谷氨酸是人体不可缺少的一种氨基酸。另外，炒菜时味精应该最后放，是因为味精在高温下会变成焦谷氨酸钠，这样就没有鲜的味道了，但根据FDA从九千多份论文总结出的结果，焦谷氨酸钠并不会致癌，只是没有提鲜的作用了。
  7. 我们到底该信谁？

中医

  1. 以下几种药最好不要吃，“藿香正气水”（可能致癌）、“牛黄解毒丸”（重金属砷）、“龙胆泻肝丸”（致癌）。
  2. 《本草纲目》这部经典，我们所熟知的李时珍尝百草，历尽30年写成。根据《卓老板聊科技》中的解释，《本草纲目》的体量，30年的时间要去搜集各种药方并进行实验去甄别其中的有效性是不可能的。《本草纲目》的序是一个文人而不是从医人员题的。我并不是否定《本草纲目》的价值—毕竟收集这么多药方是个巨大且艰辛的工程，只是我们应该客观、理性看待它的真正实用价值
  3. 屠呦呦获得诺贝尔医学奖，是表示诺贝尔奖颁给了中医吗？不是！诺奖颁奖委员会在颁奖后的记者会上明确表示，我们没有把奖颁给传统医学。屠呦呦是受传统医学的启发（指黄花蒿这种植物），使用科学的方法，经过大量实验最终在黄花蒿中成功萃取了有效治疗疟疾的青蒿素。

食品

对待保健品的态度，如果有明显的特别挑食、厌食，导致缺少维生素，可以买点维生素片补充一下。市面上绝大多数保健品都没有宣传的功效，如果价格和平常吃的蔬菜水果价格差不多，买点吃也无妨，因为无非就是些淀粉、蛋白质，不会有多少害处。如果价格很贵，根本不值得买。

    抽检40%不合格》    . 该文章引起人们对进口奶粉质量的质疑，该如何选择奶粉呢？ 只说结论：买奶粉时选择国行进口奶粉，注意，不是国产奶粉，也不是自己海淘的奶粉。–信息来自《得到》app上《卓老板聊科技》的问答部分。
  2. 冬虫夏草不能吃。冬虫夏草实际是真菌进入动物体内，动物感染死亡后，真菌长出动物体外，所以呈现出冬天虫夏天草的稀奇景象，不要被忽悠。不能吃原因有：1. 很多厂家为多赚钱不惜向虫草中掺入重金属砷。2. 并没有宣传的抗癌、抗衰等功效。但价格贵的要死。一则小故事，谷开来童鞋在公公葬礼上晕倒，就是因为这玩意吃多了，后来还非认为是被别人陷害，找了个背锅的。
  3. 阿胶性价比很低，且无宣传的那么好。且不说厂家是否真舍得拿真驴皮制作阿胶，不管猪皮还是驴皮，营养成分都是胶原蛋白—一种氨基酸成分不是很全面的蛋白质。对身体没那么大好处。（2016年1月份，深圳一位市民自费检验了自己在同仁堂买的阿胶，结果是没检测出驴、马皮成分，反而有牛、猪的DNA。）

百度指数爬取工具(BDIndexSpider)是开发出来给在做科研的女朋友用的，顺道应付毕业，做出点实用性强的软件才是最重要的。软件开发基本经历如下几个阶段：

  1. 写代码

    2016年3月初构思，开始写代码，三个星期后，完成了大体框架(听着好像挺大的工程似的，其实很菜)，基本可以正常运行实现百度指数的精确爬取。

    3月的最后一个星期，写毕业论文，自己看了都受不了，完全不是写论文的料。

    4月的大部分时间，在改论文的挣扎中度过，最终版仍然很烂。 至于软件，这段时间不间断地优化了些大小的bug。同时尝试加入了效率更高的估算模式。
